# GitTutorial

## Переміщення по проекту

```
1. pwd — Перегляд вашого поточного розташування;
2. ls — список папок та файлів у поточній директорії, де було виконано команду;
3. ls -a — список відкритих та прихованих папок та файлів у поточній директорії, де було виконано команду;
4. cd ~ — перехід до домашньої директорії поточного користувача;
5. cd .. — Перехід на один рівень вгору в ієрархії файлової системи;
6. cd folder_name — перехід до вибраної папки;
7. mkdir folder_name — створити папку із вказаним ім'ям.
```

## Налаштування Git

```
1. git config --global user.name "Name Surname" — задаємо ім'я для комітів
2. git config --global user.email "your@email" — задаємо email для комітів
3. git config --global color.ui auto — включаємо кольоровий інтерфейс у терміналі
4. git config --list — переглядаємо всі налаштування git
```

## Робота з репозиторієм

```
5. git init — ініціалізація нового репозиторію в поточній папці
6. git clone url — копіює репозиторій (папку з файлами) на комп'ютер
7. git remote -v — переглядаємо список підключених репозиторіїв
8. git remote add origin url — додаємо віддалений репозиторій
9. git remote remove origin — видаляємо віддалений репозиторій
```

## Робота з файлами та комітами

```
10. git add . — додаємо всі зміни до індексу
11. git add file_name — додаємо конкретний файл до коміту
12. git commit -m "commit message" — створюємо коміт з описом
13. git commit -am "commit message" — додаємо всі змінені файли та комітимо одним рядком
14. git commit --amend -m "New commit message" — змінюємо останній коміт
```

## Робота з гілками

```
15. git branch — показуємо список гілок в проекті
16. git branch "branchName" — створює нову гілку
17. git checkout "branchName" — переходимо на вказану гілку
18. git checkout -b "branchName" — створюємо нову гілку і переходимо на неї
19. git branch -d branchName — видаляємо локальну гілку
20. git push origin --delete branchName — видаляємо гілку з GitHub
21. git merge "branchName" — зливаємо гілку "branchName" в поточну гілку
22. git merge --abort — відміняємо злиття гілок
```

## Оновлення репозиторію

```
23. git push — відправляємо зміни на GitHub
24. git pull — отримуємо останні зміни з GitHub
25. git fetch — отримуємо зміни без злиття
```

## Перегляд змін

```
26. git status — показуємо статус проекту
27. git diff — показує відмінності між поточним станом файлів і останнім комітом
28. git diff file_name — показує відмінності у конкретному файлі
29. git diff commit_hash — показує відмінності між вказаним комітом і поточним станом файлів
30. git diff --staged — показує зміни, які вже були додані в індекс
```

## Робота з історією

```
31. git log — показує історію комітів
32. git show commit_hash — переглядаємо детальну інформацію про коміт
33. git reflog — переглядаємо історію змін у HEAD, навіть після reset
```

## Скасування змін

```
34. git restore file_name — повертаємо файл до останнього збереженого стану
35. git restore --staged file_name — прибираємо конкретний файл з індексу
36. git reset HEAD file_name — скасовуємо додавання файлу до коміту
37. git reset --soft commit_hash — повертаємося до вказаного коміту, залишаючи зміни
38. git reset --hard commit_hash — повертаємося до вказаного коміту та видаляємо всі зміни
39. git revert commit_hash — скасовуємо зміни конкретного коміту, створюючи новий коміт
```

## Додаткові команди

```
40. git stash — зберігаємо незбережені зміни в буфер обміну
41. git stash apply — вставляємо збережені зміни з буфера обміну
42. git cherry-pick commit_hash — переносимо зміни з одного коміту в іншу гілку
43. git rebase branchName — змінюємо базову гілку для поточної гілки
44. git rebase --abort — скасовуємо операцію rebase
45. git rebase --continue — продовжуємо rebase після вирішення конфліктів
46. git tag tag_name — створюємо тег
47. git tag — показуємо список тегів
48. git push origin tag_name — відправляємо тег на GitHub
49. git push --tags — відправляємо всі теги
50. git tag -d tag_name — видаляємо тег локально
51. git push origin --delete tag_name — видаляємо тег з GitHub
52. git help command_name — переглядаємо довідку по команді
```

> **Примітка:** Для того щоб вийти з режиму перегляду комітів використовуйте клавішу `q` (стосується команд №31 і №32).
